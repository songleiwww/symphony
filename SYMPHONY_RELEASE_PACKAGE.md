## 🏗️ 技术架构

### v0.7.0 三层架构

```
┌─────────────────────────────────────────┐
│         决策层 (Decision Layer)          │
│  - 任务理解与分解                       │
│  - 模型选择与分配                       │
│  - 流程编排与控制                       │
└─────────────────────┬───────────────────┘
                      │
┌─────────────────────▼───────────────────┐
│        协作层 (Collaboration Layer)      │
│  - 多模型接力模式                        │
│  - 多模型并行模式                        │
│  - 多模型评审模式                        │
│  - 自定义协作模式                        │
└─────────────────────┬───────────────────┘
                      │
┌─────────────────────▼───────────────────┐
│        执行层 (Execution Layer)          │
│  - 真实模型调用                          │
│  - 故障容错处理                          │
│  - Token统计                             │
│  - 结果整合                              │
└─────────────────────────────────────────┘
```

### 真实模型调用架构（用户方法）

```
用户对话
    ↓
生成临时Python脚本（包含真实模型调用代码）
    ↓
运行脚本（exec工具）
    ↓
输出到JSON文件
    ↓
读取JSON文件（read工具）
    ↓
清理临时文件
    ↓
展示结果给用户
```

## 📖 使用指南

### 快速开始（3步）

1. **安装依赖**
   ```bash
   pip install requests
   ```

2. **配置OpenClaw**
   - 确保 `openclaw.cherry.json` 在正确位置
   - 配置模型API Key

3. **运行真实调用**
   ```bash
   python simple_real_call_ascii.py
   ```

### 核心文件快速参考

| 文件 | 用途 |
|------|------|
| config.py | 模型配置（18个模型） |
| real_model_caller.py | 真实模型调用器 |
| symphony_real_orchestrator.py | 真实模型编排器 |
| model_manager.py | 模型管理器（含故障容错） |
| symphony_core.py | 统一调度核心 |
| memory_system.py | 记忆系统 |
| symphony_ui.py / symphony_enhanced_ui.py | 美观UI系统 |
