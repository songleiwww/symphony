# 序境系统 - AI Agent 调度系统

## 系统概述

序境系统是一个基于 AI Agent 的智能调度系统，负责协调多模型的工作，实现智能任务分发和对话接管功能。

## 核心功能

### 1. 多模型调度
- 支持多个 AI 模型服务商（魔力方舟、英伟达、硅基流动、阿里云百炼等）
- 智能模型选择与负载均衡
- 健康检查与故障转移

### 2. 对话接管
- 基于关键词触发（"序境"、"陆念昭"）
- 自动识别用户意图
- 无缝切换对话上下文

### 3. 官署体系
- 模拟古代官制架构
- 角色与模型一一绑定
- 职责明确，分工协作

## 数据库结构

### 核心表

1. **模型配置表**
   - 存储所有 AI 模型的配置信息
   - 字段：模型名称、服务商、API 地址、API 密钥、在线状态等
   - 当前模型数量：564 个
   - 服务商数量：7 个

2. **官署表**
   - 存储官署机构信息
   - 字段：名称、级别、职位、官品、职责、状态
   - 当前官署数量：63 个
   - 核心官署：少府监、中尚署、左尚署、右尚署、织染署等

3. **官署角色表**
   - 存储角色信息，与模型配置表一对一绑定
   - 字段：姓名、官职、职务、所属官署、职责、角色等级、状态、模型配置表_ID
   - 当前角色数量：根据实际数据
   - 代表角色：陆念昭（少府监）

4. **Skill 持久化表**
   - 存储 Skill 的配置和状态
   - 字段：名称、描述、核心规则、状态、版本、创建时间、更新时间
   - 核心 Skill：序境系统

5. **内核规则表**
   - 存储系统内核规则
   - 字段：规则名称、规则内容、规则类型、优先级、状态
   - 当前规则数量：10 条

6. **接管状态表**
   - 存储当前接管状态
   - 字段：状态、当前角色、参与角色、任务描述、启动时间、更新时间

7. **调度历史表**
   - 存储调度历史记录
   - 字段：timestamp、角色、任务、状态、结果

8. **任务表**
   - 存储任务信息
   - 字段：任务名称、任务描述、状态、创建时间、完成时间

9. **记忆表**
   - 存储对话记忆
   - 字段：内容、类型、access_count、last_accessed

10. **Token 认证表**
    - 存储 Token 认证信息
    - 字段：token、expires_at、is_active

## 使用说明

### 触发序境系统

```
用户：序境
或
用户：陆念昭
```

系统会自动接管对话，并调度合适的模型进行处理。

### 添加新角色

1. 在官署表中确认官署存在
2. 在模型配置表中选择合适的模型
3. 在官署角色表中添加角色，绑定模型配置表_ID

### 添加新模型

1. 在模型配置表中添加新记录
2. 填写完整信息（模型名称、服务商、API 地址、API 密钥等）
3. 设置在线状态
