### 事件总线架构师 (设计事件总线集成方案)
作为事件总线架构师，我为您设计了一套基于 **适配器模式** 和 **发布/订阅模式** 的集成方案。

该方案旨在 `symphony` 内部建立一个独立的事件生态系统，通过 `OpenClawAdapter` 将 OpenClaw 的外部调用转换为 Symphony 内部标准事件，从而实现解耦。

### 1. 设计概述与 JSON Schema 定义

首先，我们需要定义 OpenClaw 与 Symphony 之间交互的“通用语言”。以下是基于 JSON Schema 的标准事件包定义：

#### `event_schema.json`
```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "Symphony Integration Event",
  "type": "object",
  "properties": {
    "event_id": {
      "type": "string",
      "description": "全局唯一事件ID (UUID)"
    },
    "event_type": {
      "type": "string",
      "description": "事件类型标识符，例如 'openclaw.task.created'",
      "pattern": "^[a-z0-9\\.\\-]+$"
    },
    "source": {
      "type": "string",
      "description": "事件来源，固定为 'OpenClaw'",
      "const": "OpenClaw"
    },
    "timestamp": {
      "type": "string",
      "format": "date-time",
      "description": "ISO 8601 格式的时间戳"
    },
    "payload": {
      "type": "object",
      "description": "实际携带的业务数据负载",
      "additionalProperties": true
    },
    "context": {
      "type": "object",
      "description": "元数据，如用户ID、会话ID等",
      "properties": {
        "request_id": { "type": "string" },
        "user_id": { "type": "string" }
      }
    }
  },
  "required": ["event_id", "event_type", "source", "timestamp", "payload"]
}
```
